Prerequisites & Guide for Study Submission

Before you begin your submission, please ensure you have the following information and files ready. Prepared data ensures a smooth and efficient submission process.

description Metadata Requirements

  • Principal Investigator contact details (Affiliation, Email, ORCID).
  • Study Title (min 25 characters) and Abstract/Description (min 60 characters).
  • Protocol Descriptions: Sample collection, extraction, chromatography/NMR, etc.
  • Associated Publications: DOI or PubMed ID if available.

cloud_upload Data File Preparation

  • Metadata Files: ISA-Tab format (Investigation, Study, Assay files).
  • Raw Data Files: Proprietary instrument formats (e.g., .raw, .d, .wiff).
  • Derived Data Files: Open formats (e.g., .mzML, .nmrML) are highly encouraged.
  • Naming Conventions: Use only alphanumeric characters, underscores, and dots. No spaces.

Next Steps to complete your submission:

1 task

Complete 'Sample' metadata table

{{ doneStepExpandedIndex === 0 ? 'expand_less' : 'expand_more' }}

Populate the sample sheet (s_MTBLSxxx.txt) with all required fields for every sample and control in your study:

  • Source Name — subject identifier or parent entity; repeat Sample Name if not applicable.
  • Characteristics[Organism] — scientific taxonomy name (e.g., Homo sapiens), not generic terms like "human" or "mouse." Pair with ontology fields (Term Source REF, Term Accession Number).
  • Characteristics[Organism part] — tissue/fluid/fraction etc. sampled (e.g., blood plasma, whole organism, cell culture); or for controls use appropriate terms (e.g., solvent).
  • Sample Name — unique per sample; typically aligns with raw data filenames.
  • At least one Factor — use the "+Factor" button to add columns describing your experimental groups (e.g., Treatment, Genotype, Timepoint).

Do not remove columns, change column order, or alter headers. We recommend adding a few samples via the online editor first (to select correct ontology terms), then downloading, completing in Excel, and re-uploading.


Sample guide →

2 task

Complete 'Assay' metadata table(s) except data file names

{{ doneStepExpandedIndex === 1 ? 'expand_less' : 'expand_more' }}

Every sample referenced in your sample sheet should be referenced in one or more 'Assay' tables. Leave Data File columns empty for now (completed in step 4). Key fields vary by technology:

LC-MS / GC-MS assays:

  • Chromatography — instrument (manufacturer + model), column model (with dimensions), column type (e.g., reverse phase, HILIC).
  • Mass spectrometry — instrument, ion source, mass analyzer, scan polarity, and m/z range.
  • Use one assay per polarity — do not mix positive and negative in the same 'Assay' sheet. Specify alternating only if using polarity switching within a single injection.

NMR assays:

  • NMR sample — tube type, solvent, pH, temperature.
  • NMR spectroscopy — instrument (with frequency), probe, number of transients, pulse sequence, magnetic field strength.

All assay types:

  • Protocol REF cells must match their section name exactly (e.g., Extraction, Chromatography, Mass spectrometry, NMR spectroscopy, Data transformation, Metabolite identification).
  • Do not remove columns, change column order, or alter headers.

Assay guide →

3 task

Upload data files

{{ doneStepExpandedIndex === 2 ? 'expand_less' : 'expand_more' }}

Upload your instrument raw files and, where available, open-format derived files via Aspera or FTP.

  • File names must contain only letters (a–z, A–Z), numbers (0–9), hyphens (-), underscores (_), or dots (.). Spaces are not accepted.
  • Raw data folders (e.g., .d, .raw, or any NMR raw data folder) must each be compressed individually as .zip before upload, click here for guidance. A single zip containing multiple folders is not accepted.
  • Accepted raw formats include: .d, .raw, .wiff + .wiff.scan, .fid, .ser, .lcd, .smp, and others.
  • Accepted derived formats include: .mzML, .mzXML, .imzML, .cdf, .nmrML, and others. Note that some formats (e.g., .mgf, .msp, .xlsx, .cef) are not accepted alone and must accompany spectrum/chromatogram data.
  • Where an instrument raw file is not available (e.g., some non-MS chromatographic detectors), it is acceptable to submit the open-format file only.

Files guide →

4 task

Add raw/derived data filenames to 'Assay' metadata table(s)

{{ doneStepExpandedIndex === 3 ? 'expand_less' : 'expand_more' }}

Enter the full path of each file in the corresponding assay columns, always prefixed with FILES/:

  • e.g., FILES/RAW_FILES/sample1.raw.zip
  • e.g., FILES/DERIVED_FILES/sample1.mzML

MS assays require at least one of Raw Spectral Data File or Derived Spectral Data File per sample. For .wiff/.wiff.scan datasets, add an extra 'Raw Spectral Data File' column so both files are referenced.

NMR assays may include three file types per sample:

  • Free Induction Decay Data File — raw NMR data (e.g., sample1.zip).
  • Acquisition Parameter Data File — acquisition parameters (e.g., acqus.txt).
  • Derived Spectral Data File — converted format if available (e.g., .nmrML).

Verify that all referenced files exist in your uploaded study folder and that paths match exactly.


Files guide →

5 task

Complete the description of your study 'Protocols'

{{ doneStepExpandedIndex === 4 ? 'expand_less' : 'expand_more' }}

Provide detailed, reproducible descriptions for each protocol section:

  • Sample collection — origin, treatment, time points, collection and storage.
  • Extraction — preparation methods, control samples (pooled QC, blanks, standards).
  • Chromatography — instrument, column, mobile phase/gradient, temperatures, flow rate, injection volume.
  • Mass spectrometry (or NMR spectroscopy) — instrument, ion source/probe, ionisation mode, m/z range, acquisition method.
  • Data transformation — software, pipelines, processing steps.
  • Metabolite identification — software, reference databases, annotation workflow.

Protocol guide →

6 task

Index 'Data Files' & run 'Study Validation'

{{ doneStepExpandedIndex === 5 ? 'expand_less' : 'expand_more' }}

After uploading, synchronise your files with the study on the Files tab:

  • Click Index Data Files to sync uploaded raw/derived data into the study.
  • Click Synchronise Metadata & Result Files to read in any uploaded metadata files (or use the Drag & Drop feature for metadata).
  • Wait for both processes to complete — status indicators will update automatically.
  • Go to the Study Validations tab and click Start Validation.

Validation guide →

7 task

Make amendments to correct any 'Validation Error'(s) & re-validate

{{ doneStepExpandedIndex === 6 ? 'expand_less' : 'expand_more' }}

Review errors and warnings across all validation sections.

  • Address all errors and as many warnings as possible, then re-run validation — it must be re-triggered after every edit. Download the validation report for a full breakdown. If you need help, contact metabolights-help@ebi.ac.uk.
  • Validation must pass without returning 'Error'(s) before the study can be promoted to 'Private' and subsequently 'Public' status. Validation 'Warning'(s) are advisory only and do not prevent study status progression.

Validation guide →

8 task

Change study status to 'Private' (peer-review access only) or 'Public', when ready

{{ doneStepExpandedIndex === 7 ? 'expand_less' : 'expand_more' }}

Once validation passes, change status from Provisional → Private to:

  • Obtain a permanent MTBLSxxx accession number (use this in your manuscript, not the temporary REQxxx).
  • Generate a private, read-only Reviewer Link to share with journals and collaborators.

Study status is changed by click 'Status [Provisional/Private]' in the top left of the study editor and making a selection from the drop-down box.


To make further edits after going Private, temporarily revert to Provisional, make changes, re-validate, and return to Private. When ready (e.g. on publication of an associated manuscript), it must be changed to 'Public'.
